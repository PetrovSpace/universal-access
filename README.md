# universal-access

Мульти-протокольный сервер доступа к заблокированным и регионально-ограниченным сайтам. Одна команда на чистом VPS — и вы получаете готовые конфиги для всех своих устройств. Без веб-панелей, без Docker, без ручной донастройки.

Приоритет проекта — обход блокировок и надёжность, **не** анонимность.

---

## 1. Что это и какие протоколы поднимаются

Скрипт превращает свежий VPS в сервер с несколькими независимыми каналами доступа, которые работают **параллельно** под `systemd`:

| # | Протокол | Порт | Назначение |
|---|----------|------|------------|
| 1 | **VLESS + Reality gRPC** (TCP) | `2053/tcp` | **ОСНОВНОЙ** путь для Happ. gRPC обходит детект почерка raw-TCP абонентским/мобильным DPI РФ (проверено вживую). Клиенту даётся `.json` со split-routing (RU-сайты напрямую). |
| 2 | **AmneziaWG 2.0** (UDP) | `51820/udp` | Обфусцированный WireGuard — на сетях, где открыт UDP. Отдельный `.conf` на устройство. |
| 3 | **VLESS + Reality** (TCP, blacklist) | `443/tcp` | **План Б** (raw-TCP). SNI — нейтральный иностранный домен. |
| 4 | **VLESS + Reality** (TCP, whitelist) | `8443/tcp` | **План Б** для мобильных РФ / ТСПУ. SNI — разрешённый российский ресурс. |
| 5 | **MTProto-прокси** (mtg, fake-TLS) | `8888/tcp` | Прокси для Telegram, маскировка под нейтральный домен. |

Опционально (по умолчанию выключено): **Hysteria2** (UDP) — для скорости и игр.

Идея простая: основной канал — **VLESS-Reality поверх gRPC** (порт 2053): на практике именно он переживает жёсткий абонентский/мобильный DPI РФ, когда raw-TCP-Reality «подключается, но данные не идут». Если gRPC где-то не пройдёт — есть AmneziaWG (UDP) и raw-TCP Reality на 443/8443 (план Б). Все пути независимы и подстраховывают друг друга.

---

## 2. Требования

- Чистый VPS с **Ubuntu 24.04 LTS** (по умолчанию) или **Debian 12**.
- Доступ **root по SSH** (`ssh root@IP`).
- Публичный IPv4 (определяется автоматически; при необходимости можно задать вручную в `config.env`).

Всё остальное (jq, curl, openssl, uuidgen, nftables и т. д.) скрипт ставит сам.

---

## 3. Запуск одной командой

Зайдите на сервер под root и выполните:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/PetrovSpace/universal-access/main/install.sh)
```

Скрипт сам склонирует репозиторий в `/opt/access/src`, всё установит, настроит и в конце выведет готовую сводку (`SUMMARY`) со ссылками, путями к файлам и командами `scp` для скачивания конфигов.

### Альтернатива — через git clone

Если хотите сначала задать свои параметры (см. раздел 4) или просто предпочитаете явный путь:

```bash
git clone https://github.com/PetrovSpace/universal-access.git
cd universal-access
sudo ./install.sh
```

> Установка **идемпотентна**: повторный запуск чинит и обновляет, но не ломает уже выданные ключи. Чтобы добавить нового человека или устройство — допишите имя в массив в `config.env` и перезапустите `install.sh`: сгенерируются только новые, существующие останутся нетронутыми.

## 4. Свои параметры через config.env

По умолчанию ничего настраивать не нужно — скрипт берёт разумные дефолты. Если хотите переопределить порты, SNI или список клиентов — скопируйте пример и отредактируйте:

```bash
cp config.env.example config.env
nano config.env
sudo ./install.sh
```

Самые полезные параметры:

```bash
# Кто пользуется VLESS-Reality и Telegram-прокси (добавление = дописать в массив).
CLIENTS=("main" "second")

# Устройства AmneziaWG: на каждое — отдельный .conf (один conf = одно устройство).
AWG_PEERS=("mac" "keenetic" "windows")

# Порты (при занятости скрипт сам возьмёт следующий свободный).
REALITY_PORT_GRPC=2053          # ОСНОВНОЙ VLESS-Reality gRPC (обходит DPI РФ)
REALITY_PORT_BLACKLIST=443      # план Б: raw-TCP VLESS-Reality (Happ)
REALITY_PORT_WHITELIST=8443     # план Б: raw-TCP (моб. РФ / ТСПУ)
AWG_PORT=51820                  # AmneziaWG (UDP)
MTPROTO_PORT=8888               # Telegram MTProto
GRPC_SERVICE_NAME=grpc          # имя gRPC-сервиса (совпадает у сервера и клиента)

# SNI для blacklist: иностранный нейтральный домен с TLS 1.3 + HTTP/2. Берётся первый валидный.
REALITY_SNI_BLACKLIST=("www.microsoft.com" "www.cloudflare.com" "www.amazon.com")

# SNI для whitelist: РАЗРЕШЁННЫЙ российский ресурс (иначе ТСПУ рвёт handshake). Первый валидный.
REALITY_SNI_WHITELIST=("gosuslugi.ru" "www.gosuslugi.ru" "www.sberbank.ru")

# Домен маскировки для MTProto fake-TLS.
MTPROTO_MASK_DOMAIN="www.cloudflare.com"

# Hysteria2 — по умолчанию выключен.
ENABLE_HYSTERIA2=false

# Публичный IP. Пусто = автоопределение.
#PUBLIC_IP=""
```

Полный список параметров с комментариями — в [`config.env.example`](config.env.example).

> **Важно про whitelist-SNI:** для мобильных операторов РФ домен whitelist-инбаунда обязан быть из реально разрешённого списка (госуслуги, крупный банк и т. п.), иначе ТСПУ не пропустит даже TLS-handshake. Скрипт проверяет, что домен отдаёт TLS 1.3 и HTTP/2, но «разрешённость» ресурса на стороне оператора — на вашей совести.

## 5. Пошаговая настройка устройств

После установки все файлы лежат на сервере в `/opt/access/clients/`. Сначала заберите нужные файлы на свою машину (точные команды `scp` печатает сам `install.sh` в конце, и они есть в `SUMMARY.txt`).

Какой файл в какое приложение:

| Файл | Приложение |
|------|-----------|
| `vless-grpc-<имя>.json` | **Happ (ОСНОВНОЙ)** — gRPC + split-routing (RU-сайты напрямую) |
| `vless-grpc-<имя>.txt` | та же gRPC-ссылка без split-routing |
| `vless-reality-<имя>.txt` | план Б: raw-TCP VLESS-Reality (443) |
| `vless-whitelist-<имя>.txt` | план Б: raw-TCP, моб. РФ / ТСПУ (8443) |
| `amneziawg-<устройство>.conf` | AmneziaVPN (один `.conf` = одно устройство; нужен открытый UDP) |
| `telegram-proxy-<имя>.txt` | Telegram (ссылка tg:// и t.me/proxy) |

### Mac

Основной путь — AmneziaWG:

```bash
scp root@IP:/opt/access/clients/amneziawg-mac.conf .
```

1. Установите **AmneziaVPN** (Mac App Store / amnezia.org). Нужен **macOS 12 и новее**.
2. В AmneziaVPN: добавить конфигурацию → импортировать `amneziawg-mac.conf` → подключиться.

Запасной путь — VLESS-Reality через **Happ**:

```bash
scp root@IP:/opt/access/clients/vless-reality-<имя>.txt .
```

1. Откройте файл, скопируйте строку `vless://...` в буфер обмена.
2. В Happ: **«+»** → **«Импорт из буфера обмена»**. QR не нужен.

> Happ — клиент на базе Xray, поэтому ест именно ссылку `vless://` из буфера. Требует **macOS 12+**.

### iOS

- **AmneziaVPN** (App Store): импортируйте `amneziawg-<имя>.conf` (для отдельного устройства заведите свой peer в `AWG_PEERS`, например `iphone`).
- Или **Happ**: скопируйте `vless://` из `vless-reality-<имя>.txt` и вставьте через **«+»** → **«Импорт из буфера обмена»**.

### Keenetic

```bash
scp root@IP:/opt/access/clients/amneziawg-keenetic.conf .
```

Нативная поддержка AmneziaWG 2.0 на Keenetic появилась только в **KeeneticOS 5.1 Alpha 3** (Канал разработчика). Есть два пути:

**(а) Новая прошивка — нативный AWG 2.0 (рекомендуется):**
1. Обновите KeeneticOS до **5.1+** через **Канал разработчика** (Управление → Общие настройки → Версия ОС → канал «Предварительная версия» / dev).
2. Интернет → Другие подключения → **WireGuard / AmneziaWG** → импортируйте `amneziawg-keenetic.conf`.
3. Отметьте подключение как **«Использовать для выхода в интернет»**.

**(б) Старая прошивка (4.2+) — только AmneziaWG 1.0:**
- Версии 4.2+ умеют только AmneziaWG 1.0 через компонент **WireGuard**. Импортируйте `.conf` в **Интернет → Другие подключения → WireGuard**. Часть параметров обфускации 2.0 при этом не применится — если не заработает, обновляйтесь до 5.1+.

> Предупреждение: **один `.conf` = одно устройство**. Нельзя одновременно использовать `amneziawg-keenetic.conf` и на роутере, и на Mac. Каждому устройству — свой peer.

### Windows

```cmd
scp root@IP:/opt/access/clients/amneziawg-windows.conf .
```

1. Установите **AmneziaVPN** для Windows (amnezia.org).
2. Добавить конфигурацию → импортировать `amneziawg-windows.conf` → подключиться.

### Telegram

```bash
scp root@IP:/opt/access/clients/telegram-proxy-<имя>.txt .
```

В файле две ссылки. Проще всего открыть на телефоне ссылку вида `https://t.me/proxy?server=...` — Telegram сам предложит включить прокси. Альтернатива — `tg://proxy?server=...&port=...&secret=ee...` для ручного импорта в приложение.

> **Важно про MTProto-секрет.** `mtg` v2 поддерживает ровно один секрет на сервер, поэтому **Telegram-прокси использует общий секрет для всех клиентов**. Файлы `telegram-proxy-<имя>.txt` различаются только именем, ссылка внутри одинаковая и рабочая. Следствие: `regen.sh <client>` перевыпускает этот общий секрет **для всех** (старые Telegram-ссылки перестают работать у обоих — нужно раздать новые). VLESS-Reality, наоборот, имеет отдельный ключ на каждого клиента и ротируется изолированно.

## 6. Если перестало работать

Каналы независимы, поэтому почините по порядку — обычно хватает первого шага.
Подробный разбор «подключается, но интернета нет» на сетях РФ (две болезни и их лечение,
развенчанные мифы) — в [`docs/ru-dpi-troubleshooting.md`](docs/ru-dpi-troubleshooting.md).

> Сначала переключись на **gRPC** (основной транспорт, `vless-grpc-<имя>.json`) — он обходит
> детект почерка raw-TCP, который чаще всего и есть причина. Критерий, что канал жив, — крупная
> загрузка (> 1 МБ) без обрыва, а НЕ «подключилось».

**Шаг 1. Сменить SNI (быстрый ремонт VLESS-Reality).**
Чаще всего «отваливается» конкретный SNI-домен, который начали резать. Поменяйте его на другой нейтральный — доступ восстанавливается:

```bash
cd /opt/access/src
sudo ./bin/set-sni.sh blacklist www.cloudflare.com
```

Скрипт проверит, что новый домен отдаёт TLS 1.3 и HTTP/2, обновит инбаунд, перезапустит Xray и перезапишет клиентские ссылки. Заберите обновлённый `vless-reality-<имя>.txt` заново.

**Шаг 2. Переключиться на whitelist-ссылку (мобильный РФ / ТСПУ).**
Если основной путь на 443 не идёт именно с мобильного интернета — используйте ссылку из `vless-whitelist-<имя>.txt` (инбаунд 8443 с разрешённым RU-SNI):

```bash
scp root@IP:/opt/access/clients/vless-whitelist-<имя>.txt .
```

При необходимости тем же `set-sni.sh whitelist <домен>` смените и whitelist-SNI на другой разрешённый ресурс.

**Шаг 3. AmneziaWG как независимый путь.**
Если VLESS режут целиком — переключитесь на AmneziaWG (UDP): импортируйте `amneziawg-<устройство>.conf` в AmneziaVPN. Это отдельный протокол на отдельном порту, его блокируют другими методами, поэтому он часто работает, когда не работает TCP-путь.

## 7. Команды управления

Все управляющие скрипты лежат в `bin/` (запускать из каталога репозитория, по умолчанию `/opt/access/src`):

```bash
cd /opt/access/src
```

**Показать сводку снова** — все ссылки, пути и команды `scp`:

```bash
sudo ./bin/info.sh
```

**Перевыпустить ключи одного клиента или устройства** (если ключ утёк). Трогает только указанного — остальные не меняются:

```bash
sudo ./bin/regen.sh <имя>      # клиент: новый VLESS-UUID + MTProto-секрет
sudo ./bin/regen.sh mac        # устройство: новые ключи AmneziaWG
```

**Сменить Reality-SNI** (см. раздел 6):

```bash
sudo ./bin/set-sni.sh blacklist <домен>
sudo ./bin/set-sni.sh whitelist <домен>
```

**Обновить Xray-core и AmneziaWG** до свежих версий, сохранив конфиги и ключи:

```bash
sudo ./bin/update.sh
```

## Подводные камни (коротко)

- **AmneziaWG DKMS на Ubuntu 24.04.** Модуль может не собраться без заголовков ядра. Скрипт сам ставит `linux-headers-$(uname -r)` и пересобирает DKMS; если и это не помогает — автоматически переключается на userspace **`amneziawg-go`**. Установка из-за этого не падает.
- **Keenetic AWG 2.0.** Нативно — только с KeeneticOS **5.1 Alpha 3** (Канал разработчика). На 4.2+ — лишь AmneziaWG 1.0 через компонент WireGuard.
- **Happ.** Импортирует `vless://` через буфер обмена («+» → «Импорт из буфера обмена»), QR не нужен. Требует **macOS 12+**.
- **whitelist-SNI.** Для мобильных РФ обязан быть из разрешённого списка, иначе ТСПУ рвёт handshake.

## Где что лежит

- Артефакты клиентов: `/opt/access/clients/` (`*.conf`, `*.txt`, `SUMMARY.txt`).
- Исходники / репозиторий: `/opt/access/src/`.
- Состояние и ключи (для идемпотентности): `/opt/access/state/`.
- Лог установки: `/opt/access/install.log`.

Скачать сразу все клиентские файлы:

```bash
scp -r root@IP:/opt/access/clients/ .
```
