## Роль и цель

Ты — senior DevOps-инженер. Создай **git-репозиторий** с набором bash-скриптов, который превращает любой свежий VPS (Ubuntu 24.04 LTS по умолчанию, Debian 12 — тоже поддержать) в **мульти-протокольный сервер для доступа к заблокированным и регионально-ограниченным сайтам**. Приоритет — обход блокировок и надёжность, НЕ анонимность. Сервер будет запускаться у дешёвых провайдеров (FirstVDS Алматы, HostVDS) и в будущем в Финляндии/Германии — скрипт обязан работать на любом из них одинаково.

Главный сценарий пользователя: купил VPS → `ssh root@IP` → выполнил **одну команду** → получил готовые конфиги для всех своих устройств. Никаких веб-панелей, никакого Docker, никакой ручной донастройки после запуска.

## Что построить — структура репозитория

```
access-server/
├── install.sh          # главный идемпотентный установщик (одна команда)
├── lib/                # модули (можно дробить install.sh на функции)
│   ├── common.sh       # детект ОС/IP/интерфейса, логирование, проверки
│   ├── amneziawg.sh
│   ├── xray.sh
│   ├── mtproto.sh
│   └── firewall.sh
├── bin/
│   ├── info.sh         # повторно печатает все ссылки/пути к конфигам
│   ├── regen.sh        # перевыпуск ключей одного клиента (если ключ утёк)
│   ├── set-sni.sh      # сменить Reality-SNI и перезапустить (быстрый «ремонт»)
│   └── update.sh       # обновить Xray/AmneziaWG до свежих версий
├── config.env.example  # все настраиваемые параметры с дефолтами
└── README.md           # на русском, точные команды запуска
```

## Архитектура сервера — что поднять

Все четыре сервиса работают параллельно на одном VPS, под systemd, без Docker.

### 1. AmneziaWG 2.0 (UDP) — основной путь
- Установить через официальный репозиторий Amnezia: пакеты `amneziawg-dkms` + `amneziawg-tools`.
- **Обязательный fallback:** если сборка DKMS-модуля падает (частая беда на Ubuntu 24.04 — не подтягиваются заголовки ядра), скрипт должен сам поставить `linux-headers-$(uname -r)` и повторить; если и это не помогает — переключиться на userspace-реализацию `amneziawg-go`. Запуск не должен падать из-за этого.
- Параметры обфускации AmneziaWG 2.0 (Jc, Jmin, Jmax, S1, S2, H1–H4 и параметры 2.0-мимикрии) генерировать со значениями по умолчанию из официальной документации Amnezia.
- Генерировать **отдельный peer на каждое устройство** из списка `AWG_PEERS` (по умолчанию: `mac`, `keenetic`, `windows`). Один конфиг AmneziaWG работает только на одном устройстве одновременно — это жёсткое требование, нельзя шарить один .conf между Mac и роутером.
- Каждый peer → отдельный `.conf` в нативном формате AmneziaWG.

### 2. VLESS + Reality на 443 (TCP) — для Happ, blacklist-режим
- Установить Xray-core официальным скриптом, настроить через systemd.
- Транспорт `tcp`, `security: reality`, `flow: xtls-rprx-vision`, `fingerprint: chrome`.
- SNI (`serverName`/`dest`) — зарубежный домен из `REALITY_SNI_BLACKLIST` (дефолт — нейтральный высоконагруженный сайт; параметр настраиваемый). Скрипт обязан **проверить** домен перед использованием: доступен, отдаёт TLS 1.3 и HTTP/2. Если нет — предупредить и взять запасной из списка.
- Выдать `vless://`-ссылку для каждого клиента из `CLIENTS`.

### 3. VLESS + Reality на втором порту (по умолчанию 8443) — для whitelist-режима
- Второй inbound в том же Xray, отдельный SNI из `REALITY_SNI_WHITELIST` — домен **из разрешённых российских ресурсов** (крупный RU-сайт/банк/госуслуги). В whitelist-режиме мобильных операторов РФ SNI обязан быть из белого списка, иначе ТСПУ не пропустит даже handshake.
- Отдельная `vless://`-ссылка на клиента. В README пометить: это «переключиться, когда на мобильном перестал работать основной».

### 4. MTProto-прокси (Telegram) — тебе и жене
- Использовать `mtg` (9seconds/mtg, v2) с режимом **fake-TLS** (маскировка под домен из `MTPROTO_MASK_DOMAIN`).
- Сгенерировать секрет на каждого клиента из `CLIENTS` и выдать ссылку `tg://proxy?server=...&port=...&secret=ee...` (и зеркало `https://t.me/proxy?...`).

### Опционально (флаги, по умолчанию ВЫКЛ — не раздувать)
- `ENABLE_HYSTERIA2=false` — добавить Hysteria2 (для скорости/игр). Если включён — поднять и выдать конфиг.

## Клиенты — фиксированный список

В начале `config.env` объявить легко редактируемые массивы:
```
CLIENTS=("main" "second")          # VLESS Reality + Telegram-прокси
AWG_PEERS=("mac" "keenetic" "windows")  # AmneziaWG
```
Учётки/ключи создаются автоматически при запуске install.sh. Чтобы добавить нового человека — пользователь дописывает имя в массив и перезапускает install.sh (скрипт должен до-генерировать только новых, не ломая существующих).

## Поведение install.sh

- `set -euo pipefail`, аккуратное логирование шагов, цветной вывод статусов.
- **Полностью неинтерактивный по умолчанию:** все параметры берутся из `config.env` (если есть) или из дефолтов; ничего не спрашивать, чтобы «одна команда» реально работала. Переопределение — через env-переменные или флаги.
- **Идемпотентность:** повторный запуск чинит/обновляет, а не дублирует и не ломает уже выданные ключи.
- **Автодетект:** публичный IP (через несколько источников с фолбэком), дефолтный сетевой интерфейс, дистрибутив и пакетный менеджер, свободные порты. Подобрать подсеть для AmneziaWG автоматически.
- Включить **BBR** и разумный sysctl-тюнинг под скорость.
- Настроить firewall (`nftables` или `ufw`): открыть только SSH, 443, 8443, UDP-порт AmneziaWG, порт MTProto (+ Hysteria, если включён). **Не заблокировать текущую SSH-сессию** (пользователь заходит под root по SSH).
- НЕ навязывать key-only SSH и не трогать вход root агрессивно (это не про анонимность; не сломать пользователю доступ). Опциональный хардненинг — закомментированной секцией.

## Доп. скрипты

- `bin/info.sh` — заново печатает SUMMARY: все ссылки и пути к файлам. Запускается в любой момент.
- `bin/regen.sh <client|peer>` — перевыпускает ключи одного клиента/peer (Reality-UUID, AmneziaWG-ключи, MTProto-секрет), перезаписывает его .conf/ссылку, перезапускает сервис. Остальные не трогает.
- `bin/set-sni.sh <blacklist|whitelist> <domain>` — меняет Reality-SNI у нужного inbound, валидирует домен (TLS 1.3 + HTTP/2), правит конфиг и перезапускает Xray. Это ключевой инструмент «быстрого ремонта»: когда конкретный SNI начинают резать, рабочее решение — сменить домен на нейтральный, и доступ восстанавливается.
- `bin/update.sh` — обновляет Xray-core и AmneziaWG до последних версий, перезапускает сервисы, конфиги сохраняет.

## Выгрузка конфигов (без QR, без панели)

Все артефакты писать в `/opt/access/clients/`:
- `amneziawg-mac.conf`, `amneziawg-keenetic.conf`, `amneziawg-windows.conf`
- `vless-reality-<client>.txt` (ссылка vless://, blacklist-инбаунд 443)
- `vless-whitelist-<client>.txt` (ссылка vless://, whitelist-инбаунд 8443)
- `telegram-proxy-<client>.txt` (ссылки tg:// и https://t.me/proxy)
- `SUMMARY.txt` — всё в одном файле + таблица «какой файл в какое приложение».

В конце install.sh вывести в stdout чистый SUMMARY: копипастные ссылки, пути к файлам и готовые команды `scp` для скачивания .conf на локальную машину. Например, подсказать пользователю строку вида `scp root@IP:/opt/access/clients/amneziawg-mac.conf .`

## Совместимость и подводные камни (учесть в коде и README)

- **AmneziaWG на Ubuntu 24.04:** DKMS-модуль может не собраться без заголовков ядра → ставить `linux-headers`, при провале — `amneziawg-go` (userspace). Обработать оба пути.
- **Keenetic:** AmneziaWG 2.0 нативно поддерживается только с KeeneticOS 5.1 Alpha 3 (Канал разработчика); версии 4.2+ умеют только AmneziaWG 1.0 через компонент WireGuard. В README дать оба пути: (а) обновить KeeneticOS до 5.1+ через Канал разработчика для нативного AWG 2.0; (б) если прошивка старая — импорт .conf в раздел «Интернет → Другие подключения → WireGuard». Предупредить: один .conf = одно устройство.
- **Happ** — клиент на базе Xray, ест ссылку `vless://` через «+» → «Импорт из буфера». QR не требуется.
- **macOS:** актуальный Happ требует macOS 12+.

## Требования к качеству кода

- Чистый, читаемый bash; функции вместо копипасты; комментарии в ключевых местах.
- Минимум внешних зависимостей; то, что нужно (`jq`, `curl`, `openssl`, `uuidgen`), ставить автоматически.
- Понятные сообщения об ошибках с подсказкой, что делать.
