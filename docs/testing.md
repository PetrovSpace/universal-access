# Тестирование из РФ

Как проверить, что туннель реально пробивает блокировки именно с российской сети.

## Что важно понимать

- **Геолокация ≠ сетевой путь.** «Я физически в РФ» не значит «мой трафик идёт через ТСПУ так
  же, как у целевого пользователя». Проверять надо с той сети, где проблема.
- **Дата-центр ≠ абонент.** ТСПУ на uplink'е хостинга обычно мягче, чем на домашнем/мобильном
  участке. Прохождение с РФ-VPS — слабое доказательство; **непрохождение** — показательно.
- **Мобильный — самый жёсткий.** Реальный тест мобильного режима — только на телефоне с
  мобильным интернетом конкретного оператора.

## Критерий успеха

НЕ «подключилось» и НЕ «порт пингуется». **Крупная загрузка (> 1 МБ) через туннель без обрыва**
+ внешний IP стал серверным на заблокированном ресурсе. Reality маскирует хендшейк даже при
мёртвом потоке — поэтому хендшейк не критерий.

## Быстрая проверка с РФ-вантажа (VPS/устройство в РФ)

Портативный headless-клиент Xray (SOCKS) + сравнение прямого и туннельного доступа —
скрипт `ru-vantage-test.sh` в корне репозитория:

```bash
./ru-vantage-test.sh 'vless://...ссылка...' https://www.instagram.com
```

Он поднимает временный SOCKS на Xray и показывает:
- прямой внешний IP и доступность заблокированного ресурса (обычно таймаут);
- через туннель: внешний IP (должен стать серверным) и тот же ресурс (должен открыться).

Для полноты добавь к ручной проверке крупную загрузку через тот же SOCKS:
```bash
curl -s --max-time 40 --socks5-hostname 127.0.0.1:10808 \
  -o /dev/null -w '%{size_download} bytes, HTTP %{http_code}\n' \
  'https://speed.cloudflare.com/__down?bytes=5000000'
```
Полные 5 000 000 байт без обрыва = поток жив (нет «заморозки 16 КБ»).
